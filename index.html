ENTRY_04 // THE_TRINITY_LAB

Lifetime Specifications

Singleton
Click to expand specification//
DEFINITION

Created once on first resolution. Reused globally for the life of the process.

WHEN TO USE

Stateless utilities, global configuration, hardware drivers, and memory caches.

builder.Services.AddSingleton<ICache, Cache>();
Scoped
Click to expand specification//
DEFINITION

One instance per unique HTTP Request. Disposed automatically on request end.

WHEN TO USE

EF Core Contexts (DbContext), User Identity services, and Repository patterns.

builder.Services.AddScoped<IRepo, DbRepo>();
Transient
Click to expand specification//
DEFINITION

A fresh instance is spawned every single time it is requested from the container.

WHEN TO USE

Lightweight, thread-safe stateless services with zero persistent state.

builder.Services.AddTransient<ILog, Log>();
